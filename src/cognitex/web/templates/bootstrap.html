Bootstrap Files

Bootstrap files define your personality, safety rules, and operational context. Edit these markdown files to customize how Cognitex operates.

Communication Style & Voice

~/.cognitex/bootstrap/SOUL.md

Define how you want Cognitex to communicate: greeting style, tone, sign-off preferences, etc.

Operator Profile

~/.cognitex/bootstrap/USER.md

Your role, relationships, and priorities. Replaces IDENTITY.md — sections with [FILL] are skipped in prompts.

Operating Constitution

~/.cognitex/bootstrap/AGENTS.md

Agent mission, principles, and safety boundaries. Section 5 (Safety) is injected as hard constraints into every agent prompt.

Tools & Infrastructure

~/.cognitex/bootstrap/TOOLS.md

SSH hosts, API services, and infrastructure details. Sections with [FILL] are skipped in prompts.

Operational Memory

~/.cognitex/bootstrap/MEMORY.md

Curated operational memory — key decisions, recurring patterns, and learned preferences. Distinct from daily memory logs (~/.cognitex/memory/).

User Context & Preferences (Legacy)

~/.cognitex/bootstrap/IDENTITY.md

Legacy file — superseded by USER.md. Kept for backward compatibility; used as fallback if USER.md doesn't exist.

Ambient Context

~/.cognitex/bootstrap/CONTEXT.md (auto-updated)

This file is automatically updated with recent activity, open threads, and relationship notes. You can also edit it manually.
